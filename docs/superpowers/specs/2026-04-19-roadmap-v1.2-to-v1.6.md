# Peshitta Constellations Roadmap: v1.2 → v1.6

**Date:** 2026-04-19
**Status:** Approved
**Author:** Jossi Fresco — ORCID [0009-0000-2026-0836](https://orcid.org/0009-0000-2026-0836)
**Concept DOI:** [10.5281/zenodo.19358529](https://doi.org/10.5281/zenodo.19358529)

## Strategic Direction

Two-axis prioritization, validated with the user:

1. **Workflow integration first** — features that let scholars *produce* and *cite*, not just browse
2. **Data depth second** — enrich the data the new workflows consume

Audio reader is removed first (no viable audio source, contracting too expensive).

Each release ships a coherent improvement scholars can immediately use. Every release inherits the citation metadata established in v1.2 (ORCID + DOI in all exports, APIs, and shareable URLs).

## v1.3 — "Open Doors"

**Theme:** Make the tool readable by other scholarly tools and AI agents.

### Features
- OpenAPI 3.0 spec generated from Flask routes (`/openapi.json`, `/docs` Swagger UI)
- MCP server (`peshitta_roots/mcp_server.py`) exposing root search, passage retrieval, concordance, cognate lookup
- USFM export endpoint: `/api/export/usfm?book=Matthew&chapter=1`
- OSIS export endpoint: `/api/export/osis?book=Matthew`
- Text-Fabric compatible export: per-book TF dataset download (`/api/export/text-fabric/<book>`)
- API documentation page (`/api`) with copy-paste examples in curl, Python, JavaScript
- Public rate limits documented (and enforced via Flask-Limiter)
- Every API response includes citation block (author, ORCID, DOI, version)

### Files
- Modify: `peshitta_roots/app.py` (add export routes, OpenAPI generator, rate limiter)
- Create: `peshitta_roots/mcp_server.py`
- Create: `peshitta_roots/exporters/usfm.py`
- Create: `peshitta_roots/exporters/osis.py`
- Create: `peshitta_roots/exporters/text_fabric.py`
- Create: `peshitta_roots/templates/api.html` (interactive docs)
- Modify: `requirements.txt` (add `flask-limiter`, `apispec`, MCP SDK)

## v1.4 — "Study Sets"

**Theme:** Collaboration and sharing without accounts.

### Features
- Named bookmark collections ("My dissertation passages", "Sermon prep — Matthew sermon series")
- Shareable collection URLs: collection encoded in URL fragment (no server storage)
  - `/bookmarks?share=<base64-encoded-collection-json>`
- Saved searches with descriptive names (localStorage)
- Annotated passages: add notes to a verse + share the annotated view
- "Send to colleague" — generates a self-contained URL with all context (collection + active script + lang)
- Import/export collections as JSON
- Each shared URL includes citation metadata (ORCID + DOI in the encoded payload)

### Files
- Modify: `peshitta_roots/templates/bookmarks.html` (collections UI)
- Modify: `peshitta_roots/static/style.css` (collection cards)
- Create: `peshitta_roots/static/js/collections.js`

## v1.5 — "On Paper"

**Theme:** Produce publishable artifacts from a research session.

### Features
- PDF export per root analysis (visualizer + cognates + concordance)
- Annotated passage PDF (verse + transliteration + translation + notes)
- Concordance report PDF (search results with full context)
- Print-optimized methodology page (clean `@media print` styles)
- Customizable export options:
  - Which transliteration script (Latin/Syriac/Hebrew/Arabic)
  - Which translation language
  - Include/exclude cognates, semantic bridges, Greek parallels
- Every PDF footer includes citation block + ORCID + DOI + access date

### Files

## v1.6 — "Deeper Wells"

**Theme:** Enrich the data that all the new workflow features consume.

### Features
- Cognate expansion 436 → 1,500+ roots
  - Use existing `scripts/generate_new_cognates.py` as starting point
  - Backfill `greek_parallel`, `sabor_raiz_*`, `hebrew[]`, `arabic[]` for new entries
- Greek concordance enrichment:
  - Index every SBLGNT lemma (not just the 368 in `greek_parallel`)
  - Statistical co-occurrence with Syriac roots at verse level
  - "Translation pattern" view per Greek lemma: top Syriac translations + consistency score (% of occurrences using the dominant Syriac root)
- Backfill `greek_parallel` field for all newly added cognates so they appear in the existing concordance index too

### Files
- Modify: `data/cognates.json` (massive expansion)
- Modify: `peshitta_roots/app.py` (`_build_greek_index` rebuilt from SBLGNT corpus, not cognates.json)
- Create: `peshitta_roots/translation_patterns.py` (statistical co-occurrence analysis)
- Modify: `peshitta_roots/templates/index.html` (translation pattern view in Greek tab)

## Cross-Cutting: Citation Metadata Everywhere

Every release inherits these standards:

- **All API responses** include a `_citation` block with author, ORCID, DOI, version, access date
- **All exports** (BibTeX, USFM, OSIS, Text-Fabric, PDF, JSON) include citation header
- **All shareable URLs** include encoded version/DOI for reproducibility
- **All download filenames** include version: `peshitta-matthew-v1.3.usfm`, `kthb-root-v1.5.pdf`

---

## Out of Scope (across all releases)

- User accounts / authentication
- Hosted collaboration (real-time editing, comments)
- Audio (removed in v1.2, no plans to revisit until viable source exists)
- More OT books beyond current 4 (Psalms, Isaiah, Ezekiel, Proverbs)
- OT translations (separate project)
- Full morphological annotation (separate, larger project)

---

## Success Criteria

By v1.6 ship:
- Scholars can cite the tool in 4 styles, with ORCID + DOI, in one click
- Other scholarly tools (Logos, Accordance, Text-Fabric, Zotero) can read its data
- AI agents can query it via MCP
- Scholars can save, share, and export their research
- The data underlying all of this covers the majority of Peshitta NT roots, not a third
