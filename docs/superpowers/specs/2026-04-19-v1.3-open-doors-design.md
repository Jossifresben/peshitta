# v1.3 Open Doors — Design Spec

**Date:** 2026-04-19
**Status:** Approved
**Roadmap:** Builds on v1.2 citation infrastructure
**Author:** Jossi Fresco — ORCID [0009-0000-2026-0836](https://orcid.org/0009-0000-2026-0836)
**Concept DOI:** [10.5281/zenodo.19358529](https://doi.org/10.5281/zenodo.19358529)

## Goal

Make the Peshitta corpus and analytical data readable by other scholarly tools (Logos, Accordance, Text-Fabric, Zotero), AI agents (via MCP), and any HTTP client (via documented OpenAPI). Every export carries the v1.2 citation metadata.

---

## 1. OpenAPI 3.0 Specification

### Approach
Hand-written OpenAPI 3.0 YAML at `peshitta_roots/static/openapi.yaml`. Auto-generation via apispec adds dependency churn for a small public API (~10 endpoints). Hand-written keeps full control of descriptions, examples, and tags.

### Routes
- `GET /openapi.yaml` — serve the YAML spec (already in static/)
- `GET /openapi.json` — serve a JSON-converted version (generated from YAML at request time, cached)
- `GET /docs` — Swagger UI page that loads `/openapi.yaml` (use the standalone Swagger UI HTML pattern, no npm dependency)

### Endpoints documented
All existing `/api/*` endpoints:
- `/api/verse`, `/api/suggest`, `/api/word-root`, `/api/roots`
- `/api/root-family`, `/api/passage-constellation`
- `/api/reverse-search`, `/api/text-search`
- `/api/concordance`, `/api/greek-concordance`
- `/api/citation`
- New v1.3 endpoints: `/api/export/usfm`, `/api/export/osis`, `/api/export/text-fabric`

### Spec metadata
The OpenAPI `info` block includes the citation metadata (contact = author + ORCID; license = Apache 2.0; x-doi = concept DOI).

## 2. USFM Export

### Endpoint
`GET /api/export/usfm?book=Matthew&chapter=1&start_verse=1&end_verse=25`

Parameters:
- `book` — required (NT or OT book name)
- `chapter` — optional (default: full book)
- `start_verse`, `end_verse` — optional (default: full chapter)

### Output
USFM 3.0 plain text with:
- `\id` line with book code (3-letter USFM book ID)
- `\h` running header
- `\toc1`, `\toc2`, `\toc3` table of contents entries
- `\c` chapter markers
- `\v` verse markers with Syriac text
- Header comment block with citation metadata (author, ORCID, DOI, version, generated date)

Returns `text/plain` with `Content-Disposition: attachment; filename="<book>-v1.3.usfm"`

### Implementation
- New module `peshitta_roots/exporters/usfm.py`
- Book code mapping (Matthew → MAT, Mark → MRK, etc.) for all 26 books in the corpus
- Strips Syriac diacritics? **No** — preserves them (USFM is Unicode-aware)

## 3. OSIS Export

### Endpoint
`GET /api/export/osis?book=Matthew`

Parameters:
- `book` — optional (default: full corpus)

### Output
OSIS XML 2.1.1 conforming document with:
- `<osisHeader>` containing work info, author (with ORCID), DOI
- `<osisText>` with `<div type="bookGroup">`, `<div type="book">`, `<chapter>`, `<verse>` elements
- Syriac text inside verses

Returns `application/xml` with `Content-Disposition: attachment; filename="<book>-v1.3.osis.xml"`

### Implementation
- New module `peshitta_roots/exporters/osis.py`
- Use Python's `xml.etree.ElementTree` (stdlib, no dependency)
- OSIS book IDs (Matthew → Matt, Genesis → Gen, etc.)

## 4. Text-Fabric Compatible Export

### Endpoint
`GET /api/export/text-fabric?book=Matthew` (returns ZIP of TF dataset for that book)
`GET /api/export/text-fabric?all=true` (returns ZIP of full corpus TF dataset)

### Output
ZIP file containing minimal Text-Fabric dataset structure:
- `otype.tf` — node types (book, chapter, verse, word)
- `oslots.tf` — slot containment
- `book.tf` — book name feature
- `chapter.tf` — chapter number feature
- `verse.tf` — verse number feature
- `text.tf` — Syriac text per word
- `root.tf` — extracted triliteral root per word
- `meta.json` — dataset metadata with citation block (author, ORCID, DOI)

Returns `application/zip` with `Content-Disposition: attachment; filename="peshitta-tf-<book>-v1.3.zip"`

### Implementation
- New module `peshitta_roots/exporters/text_fabric.py`
- Use Python's `zipfile` (stdlib)
- TF feature files are simple line-based text format — no TF library dependency needed for export

### Scope decision
Export only — we do NOT add TF as a runtime dependency. Scholars who want to consume the export use their own TF installation.

## 5. MCP Server

### Approach
Standalone Python script `peshitta_roots/mcp_server.py` using the official Anthropic MCP Python SDK (`pip install mcp`). Runs as a separate process (not part of the Flask app) to keep the web app's dependency surface clean.

### Tools exposed
- `peshitta_search_root(root)` → root forms, references, cognates
- `peshitta_get_verse(reference, lang)` → Syriac + transliteration + translation
- `peshitta_concordance(form)` → all occurrences with KWIC context
- `peshitta_greek_concordance(greek_word)` → Syriac roots translating it
- `peshitta_passage(book, chapter, v_start, v_end)` → roots in passage with cognates
- `peshitta_cite(view, label)` → citation in 5 styles

### Each tool response includes
A `_citation` field with author, ORCID, DOI, version (matches the v1.2 metadata baseline).

### Documentation
README section in repo explaining how to add the MCP server to Claude Desktop / Claude Code config.

### Scope decision
The MCP server reads the same data files as the Flask app (cognates.json, corpus CSVs). It does NOT make HTTP calls to the live Flask app — it works offline against the local data.

## 7. Rate Limiting

### Approach
Use `flask-limiter` with these defaults:
- All `/api/*` endpoints: 60 requests/minute per IP
- Export endpoints (`/api/export/*`): 10 requests/minute per IP (heavier responses)
- Citation endpoint (`/api/citation`): 120 requests/minute (small response, common use)

Rate limits are headers-based (`X-RateLimit-*`), 429 response when exceeded.

### Storage
Memory-only (single-process Render instance). For multi-process scaling, would need Redis but not v1.3 scope.

### Documentation
Limits documented in OpenAPI spec and `/api` page.

---

## 8. Citation in API Responses

Every export endpoint and the existing `/api/citation` endpoint include the metadata baseline. For non-citation endpoints, we add an optional `_citation` field shown when `?include_citation=true` is passed (avoids bloat by default).

USFM, OSIS, and Text-Fabric exports include the citation in their natural metadata sections (USFM header comment, OSIS `<osisHeader>`, TF `meta.json`).

## Out of Scope (v1.3)

- Auto-generating OpenAPI from Flask code (decided: hand-written)
- Apispec/Flask-Smorest dependency
- Real-time/streaming endpoints (WebSocket, SSE)
- API authentication / API keys (this is a public read-only API)
- Multi-process rate limiting (Redis-backed)
- USFM/OSIS export of pre-extracted roots metadata (just the text + structure)

9. MCP `peshitta_search_root` tool returns data with `_citation` block
10. All export filenames include `v1.3` and download with proper Content-Disposition
